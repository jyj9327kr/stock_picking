# Korean Market Quant Trading Pipeline

🔗 **[웹 대시보드 바로가기 (Live Dashboard)](https://jyj9327kr.github.io/stock_picking/)**

한국 주식시장(KOSPI / KOSDAQ)에서 대세 상승 국면의 안전한 주도주를 골라내기 위한 파이썬 기반 정량적(Quant) 주식 스크리닝 파이프라인 시스템입니다. 

Mark Minervini의 **8대 추세 템플릿(Trend Template)** 기술적 필터와 실시간 금융감독원 **오픈 DART API** 재무 분석을 결합하여 강력한 모멘텀과 탄탄한 펀더멘탈을 겸비한 "Safe Market Leaders"를 선별합니다.

---

## 🛠️ 주요 특징

### 1. Minervini 8대 기술적 필터 직접 적용 (Stage 1)
* **현재가 위치**: 현재가 > 150일선 및 200일선
* **이동평균 정배열 1**: 150일선 > 200일선
* **200일선 상승 추세**: 현재 200일선 > 1개월 전(20영업일 전) 200일선 (상승 추세 공고화)
* **이동평균 정배열 2**: 50일선 > 150일선 및 200일선
* **단기 지지 지표**: 현재가 > 50일선
* **신저가 탈출**: 현재가 >= 52주 신저가 * 1.3 (바닥에서 최소 30% 반등 확인)
* **신고가 근접**: 현재가 >= 52주 신고가 * 0.75 (고점 대비 25% 이내 유지)
* **지수 대비 상대 강도**: 개별 종목 3M 수익률 > 시장 기준 3M 수익률
*(데이터 신뢰성 확보를 위해 상장 250영업일 미만의 신규 상장주는 자동 배제됩니다.)*

### 2. 통합 대세 상승 벤치마크 지수 적용
* KOSPI 지수와 KOSDAQ 지수의 3개월 누적 수익률 중 **더 높게 상승한 지수를 일괄 벤치마크**로 채택합니다.
* 시장이 상승하거나 특정 지수만 강세를 보일 때 덜 하락했다는 이유로 통과하는 착시 주식을 배제하고, 두 시장 지수 중 최강자를 능가하는 진정한 모멘텀 리더만 엄격하게 걸러냅니다.

### 3. 주도 업종 실시간 집계
* Stage 1을 통과한 종목들의 한국거래소(KRX KIND) 업종 분류 데이터를 분석하여, 가장 높은 지분과 강세를 차지하고 있는 **상위 3대 섹터**를 자동으로 추출하여 리포트 상단에 시각화해 줍니다.

### 4. 오픈 DART API 연계 실시간 재무 검증 및 리스크 진단 (Stage 2)
* 1일 10,000건 제한인 DART API 쿼리를 보존하기 위해, **Stage 1 기술적 조건을 모두 통과한 종목 전체**에 대해서만 DART 실시간 재무 분석을 적용합니다.
* 연결재무제표(CFS)와 개별재무제표(OFS)를 안전하게 순회하며 아래 **3대 핵심 재무 요건**을 연산하고 **4대 다차원 리스크 요인**을 진단하여 `Risk_Flags` 컬럼에 표기하는 **진단 모드(Diagnostic Mode)**로 작동합니다.

#### 📈 3대 핵심 재무 지표 연산:
* **ROE (수익성)**: 최신 분기 또는 TTM ROE >= 15% `(당기순이익 / 자본총계) × 100`
* **EPS YoY (성장성)**: 전년동기 대비 분기순이익 증가율 >= 15% `((당기순이익 - 전기동기순이익) / |전기동기순이익|) × 100`
* **EPS QoQ (안정성)**: 직전 분기 대비 순이익 증가율 >= 0% `((당기순이익 - 직전분기순이익) / |직전분기순이익|) × 100` (단기 역성장 배제 여부 판단용)

#### ⚠️ 4대 리스크 진단 요인 (`Risk_Flags`):
1. **영업이익 vs OCF 괴리**: 장부상 영업이익은 흑자이지만, 실제 들어온 돈인 영업활동현금흐름(OCF)은 적자인 경우.
2. **FCF(잉여현금흐름) 적자**: `영업활동현금흐름(OCF) - 유형/무형자산 취득액(CAPEX) < 0`으로 실제 쓸 수 있는 잉여 현금이 고갈 상태인 경우.
3. **운전자본 급증 (매출채권 & 재고자산)**:
   * 매출 성장률이 5% 이상일 때, 매출채권 또는 재고자산 증가율이 매출 성장률의 2배를 초과하는 경우.
   * 매출 성장률이 5% 미만(또는 역성장)일 때, 매출채권 또는 재고자산 증가율이 15% 이상으로 치솟는 경우.
4. **자본 희석 위험 (CB / BW 및 유상증자 이력)**: 최근 2개년(전년도 및 금년도) 동안 주주 가치를 희석시키는 전환사채(CB), 신주인수권부사채(BW), 유상증자, 제3자배정 이력이 2회 이상 발생한 경우.

### 5. 로컬 디스크 캐싱
* 일봉 데이터(1일 유효), 종목 리스트, DART 공시 데이터(30일 유효) 등을 로컬 디스크 캐시(`.pkl`)로 자동 관리하여, 반복 실행 시의 속도를 극대화하고 API 사용량을 보존합니다.

## 🚀 실행 방법

### 1. 환경 설정 및 의존성 설치
`.env` 파일을 루트 디렉토리에 생성하고 발급받은 DART API 키를 지정합니다.
```env
DART_API_KEY="여러분의_오픈_DART_API_키"
```

필요한 패키지를 가상환경 등에 설치합니다:
```bash
pip install -r requirements.txt
```

### 2. 스크리너 파이프라인 구동

```bash
# [매일] 장 마감 후 - Stage 1 (기술적 필터링 및 주도 섹터 3개 추출)
python main.py --schedule daily

# [매주] 주말 - Stage 1 + Stage 2 (실시간 DART 재무 요건 및 리스크 진단 통합 스크리닝)
# 최근 4주일 분량의 히스토리 분석 결과가 weekly_results 폴더에 자동 누적 생성됩니다.
python main.py --schedule weekly

# [분기] 실적 공시 시즌 - DART 재무 데이터 대량 업데이트 및 캐싱
python main.py --schedule quarterly
```

## 📊 결과물 및 대시보드 연동 안내

### 1. 생성 파일 구조
* **`output/` 하위 파일**: 각 실행 타임스탬프와 함께 로컬에 보관되는 마크다운 보고서(`.md`)와 스프레드시트용 데이터(`.csv`) 파일입니다.
* **`weekly_results/` 하위 파일**: 주간 단위 실행 데이터(`.csv`) 및 웹 대시보드 시각화용 통합 데이터 파일(`data.json`)이 보관됩니다. `data.json`은 최근 4주간의 스크리닝 내역을 집계하여 대시보드에서 해당 종목이 몇 번 등장했는지(`Appearance_Count`) 추적하는 용도로 쓰입니다.

### 2. 출력 컬럼 정보 설명
* **`Ticker`**: 종목 코드 (6자리)
* **`Name`**: 종목명
* **`Sector`**: KRX 표준 업종 분류
* **`Current_Price`**: 당일 종가
* **`Marcap`**: 시가총액
* **`Revenue` / `Operating_Income`**: 당기 분기 매출액 / 영업이익 (단위: 억원)
* **`ROE`**: 당기 ROE (%)
* **`Excess_Return_3M`**: 지수 대비 3개월간의 초과 수익률 (%)
* **`Risk_Flags`**: 재무 위험 진단 결과 (`"정상"` 또는 검출된 리스크 리스트 쉼표 구분 표기)
